This skill produces distinctive, production-grade frontend interfaces. Every output must be functional code with a clear aesthetic point-of-view — not a template with placeholders, not a wireframe with generic colors, not a design that could have been generated by any AI on any project.

**Core principle**: the best visual designs are opinionated. A bold minimalist and a maximalist baroque piece are both valid — what is not valid is the comfortable middle ground that looks like everything else.

---

## Step 1 — Commit to a direction before writing a single line

Answer these four questions explicitly. Do not start coding until each has a real answer (not a generic one).

| Question | What a bad answer looks like | What a good answer looks like |
|----------|------------------------------|-------------------------------|
| **Purpose** — What problem does this solve? Who uses it? | "A website for a company" | "A portfolio for a motion designer who wants to feel like the work is already moving before you click anything" |
| **Tone** — Pick a single aesthetic extreme | "Modern and clean" | "Brutalist editorial — raw grid, oversized serif headlines, visible structure, no softness" |
| **Constraints** — Framework, performance, a11y, existing design system? | (skipped) | "React + Tailwind, must be accessible, no external font requests in production" |
| **Differentiation** — What is the one thing someone will remember? | "Nice animations" | "The cursor leaves a ink-bleed trail that matches the dominant color of the hovered section" |

Tone options for inspiration — pick one and push it to its logical extreme:
- Brutally minimal · Maximalist chaos · Retro-futuristic · Organic/natural · Luxury/refined · Playful/toy-like · Editorial/magazine · Brutalist/raw · Art deco/geometric · Soft/pastel · Industrial/utilitarian · Cyberpunk/neon · Swiss modernist · Memphis/pop

**Expected output**: a one-paragraph design brief written before any code — tone, key visual decisions, and the one unforgettable thing.

## Step 2 — Establish the visual foundation

Define these explicitly before building components. Use CSS custom properties for all tokens.

### Typography
- Pair a **display font** (for headings, hero text) with a **body font** (for reading). They should create tension, not blend together.
- Minimum type scale: 5 sizes. Use a ratio (1.25 — Major Third, 1.333 — Perfect Fourth, 1.618 — Golden ratio).
- Avoid: Inter, Roboto, Arial, system-ui, Space Grotesk as defaults. These are valid if the aesthetic demands neutrality — but make that a conscious choice, not the default.
- Good pairing strategies: geometric sans + humanist serif · condensed display + wide body · monospace + elegant serif · expressive script headline + neutral body

```css
:root {
  --font-display: 'Playfair Display', serif;   /* distinctive, characterful */
  --font-body: 'DM Sans', sans-serif;           /* clean, readable contrast */
  --text-xs: clamp(0.75rem, 1vw, 0.875rem);
  --text-sm: clamp(0.875rem, 1.2vw, 1rem);
  --text-base: clamp(1rem, 1.5vw, 1.125rem);
  --text-lg: clamp(1.25rem, 2vw, 1.5rem);
  --text-xl: clamp(1.75rem, 3vw, 2.25rem);
  --text-2xl: clamp(2.5rem, 5vw, 4rem);
  --text-display: clamp(4rem, 10vw, 9rem);
}
```

### Color
- Commit to a dominant color + 1 sharp accent + neutral scale. Three colors used with conviction beat eight used timidly.
- Avoid: purple gradient on white, teal on dark navy, flat pastel rainbow. These are the default AI palette.
- Use `hsl()` variables so tints and shades derive from a single hue:

```css
:root {
  --hue: 24;
  --color-primary: hsl(var(--hue) 90% 52%);
  --color-primary-dim: hsl(var(--hue) 90% 38%);
  --color-surface: hsl(var(--hue) 15% 8%);
  --color-surface-raised: hsl(var(--hue) 12% 13%);
  --color-text: hsl(var(--hue) 10% 92%);
  --color-text-muted: hsl(var(--hue) 8% 55%);
  --color-accent: hsl(calc(var(--hue) + 180) 80% 58%);
}
```

### Spatial composition
- Default to **unexpected layouts**: asymmetry, overlap, diagonal flow, grid-breaking elements, full-bleed sections interrupted by inset content.
- Negative space is not emptiness — it is weight. Generous negative space can be the visual statement.
- Avoid: centered card on white background, evenly spaced feature grid, top nav + hero + 3 columns + footer.

**Expected output**: CSS custom properties file or `:root` block with all tokens defined.

## Step 3 — Motion and depth

Every animation must earn its place. No animation purely because it's possible.

### When to animate
| Trigger | Good use of animation | Bad use of animation |
|---------|----------------------|---------------------|
| Page load | Staggered reveals that build narrative (`animation-delay`) | Everything fading in simultaneously |
| Hover | State change that confirms interactivity | Spinning, bouncing, scaling >1.05 |
| Scroll | Parallax depth, progressive reveal of content | Elements flying in from all directions |
| Interaction feedback | Micro-animation confirming an action | Decorative loop unrelated to the action |

### Technical rules
- Easing: `cubic-bezier(0.16, 1, 0.3, 1)` (expo-out) for entrances · `ease-in` for exits · never `linear` for UI motion
- Duration: 150ms micro · 300–400ms component · 600–800ms page/hero. Nothing above 1s unless it is the hero statement.
- Use `transform` and `opacity` only — never animate `width`, `height`, `top`, `left` (causes layout reflow)
- Always include `@media (prefers-reduced-motion: reduce)` — disable all non-essential transitions

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

### Depth and atmosphere
Create surfaces, not flat canvases. Techniques by aesthetic:
- **Dark/moody**: radial gradient backgrounds, inner glow on hover, layered box-shadows with color tint
- **Editorial/stark**: sharp drop shadows (no blur), inked borders, texture overlays (`background-blend-mode: multiply`)
- **Organic/soft**: noise texture via SVG filter or CSS, blob shapes with `border-radius` extremes, gradient meshes
- **Luxury/refined**: very subtle grain overlay (2–4% opacity), thin borders, large amounts of negative space, precise kerning (`letter-spacing`)

## Step 4 — Implementation standards

Regardless of aesthetic direction, the code must meet these standards:

- **Responsive**: use `clamp()` for fluid type and spacing — no breakpoint-only sizing for core elements
- **Accessible**: minimum contrast 4.5:1 for body text, 3:1 for large text and UI components; focus states never removed without replacement
- **Performant**: images with `width`/`height` to prevent CLS; fonts with `font-display: swap`; no layout-triggering CSS animations
- **Self-contained**: a component must work when dropped into any page — no global style dependencies unless explicitly documented

## Anti-patterns — with their fix

| Anti-pattern | Why it fails | Fix |
|---|---|---|
| Inter / Roboto as default | Signals "I didn't choose a font" | Pick a font that fits the tone; use a neutral sans only if neutrality *is* the aesthetic statement |
| Purple gradient on white | The most overused AI color scheme | Commit to a single hue with `hsl()` variables; derive the palette from it |
| Centered card, white background, drop shadow | Zero visual identity | Start with the background, not the card; the background IS the design |
| Equal visual weight on everything | No hierarchy, user doesn't know where to look | One element dominates. Everything else is secondary or tertiary. |
| Animations on every element | Exhausting, dilutes impact | One hero animation done exceptionally > ten mediocre micro-animations |
| `height: 100vh` hero with centered text and a CTA | The default AI landing page | Break the grid. Put text and image in unexpected spatial relationship. |
| Generic icon library, default size, default color | Filler, not design | Use icons sparingly; if used, match weight/style precisely to the typography |
| `box-shadow: 0 4px 6px rgba(0,0,0,0.1)` | The default shadow that says nothing | Make shadows colored (tinted to the surface hue) and purposeful |

## Definition of Done

The output is complete when **all** of the following are true:

- [ ] A design brief exists (tone, key visual decision, the one unforgettable thing)
- [ ] Typography uses a non-generic display + body pairing with a fluid scale via `clamp()`
- [ ] Color palette is defined with `hsl()` CSS variables, not hardcoded hex values
- [ ] The layout is not a standard top-nav + hero + grid — it has a deliberate spatial statement
- [ ] At least one motion effect is present and intentional (with `prefers-reduced-motion` respected)
- [ ] Contrast ratio ≥ 4.5:1 on all body text
- [ ] Focus states are visible on all interactive elements
- [ ] No font from the banned list (Inter, Roboto, Arial, system-ui, Space Grotesk) unless the choice is explicitly justified by the aesthetic
- [ ] The design could not have been produced for a different project — it is specific to this context
